
### 阶段一：基础设施先行（没有这层，后面全是空中楼阁）

**1. 多场所市场数据基础设施**

- 接 Binance/OKX/Bybit 的 L2 WebSocket，构建本地实时 OB 镜像
- 解决时延对齐、断线重连、序号校验、增量更新和快照同步
- 测量端到端时延，知道你的"现在"比交易所真实"现在"晚多少毫秒
- 建立 tick 数据归档管道，为后续回测和参数校准储备数据

**2. 订单管理系统（OMS）**

- 完整的订单状态机：Pending → Active → PartiallyFilled → Filled / Cancelled / Rejected
- 处理 ghost fill：撤单消息在途期间的成交回报
- 库存实时同步：每个 fill 必须在毫秒级更新本地 q
- 批量挂撤单的原子化操作，避免半成功状态

**3. 配对的 paper trading 环境**

- 与生产环境完全一样的代码路径，但订单走模拟撮合
- 用真实市场数据驱动撮合，但不发真实订单
- 这是你检验逻辑正确性的安全网

这个阶段不涉及任何做市策略，但**80% 的实盘事故来自这一层的缺陷**。

### 阶段二：定价与流动性供给（GLT 真正发挥作用的地方）

**4. 公平价值引擎**

- 参考所 micro-price 计算（前 N 档量加权）
- Basis 滚动估计（perp - spot 的 TWAP）
- Funding rate 折算成 carry 成本
- 多参考所价格融合（如果用多个 reference venue）
- 价值的延迟补偿

**5. GLT 价差引擎**

- 参数 σ、A、k 的滚动校准（每分钟刷新）
- γ 和 Q 的初始设定与回测优化
- 输出 δᵃ(q)、δᵇ(q) 与对应的 skew

**6. Ladder 形态引擎**（GLT 不管的部分）

- 档数 N、档间距 Δ 的选择
- 每档量的分配规则（参考所深度比例 / 资本预算 / 库存乘数）
- 库存感知的非对称缩放

**7. LOB 结构感知层**（你直觉里的墙问题）

- 实时扫描参考所深度，识别流动性聚集点
- GLT 给出的 δ 作为价差下限，最终落点对齐到结构性位置
- 队列优先级估计：你在每个挂单档位的预期排队深度

### 阶段三：风险管理与执行优化（让策略能活下去）

**8. 库存对冲层**

- 在 Binance 上对冲目标所积累的库存（如果是跨所做市）
- 对冲成本（taker fee + 滑点）的实时计算
- 对冲触发阈值与频率优化

**9. 逆向选择监控**

- 实时计算 fill 后的 mid 反向移动幅度
- 识别"毒性流量"特征：连续同向 fill、单笔大量 fill 前后的价格变化
- 触发条件下临时拉宽 spread 或暂停报价

**10. 极端市场状态检测**

- σ 突变检测（实时波动率 / 长期波动率比值）
- 资金费率异常监控
- Order book 异常监控（深度骤减、点差骤宽）
- 各级别的自动响应：减仓、拉宽 spread、单边关停、完全 kill

**11. 风控硬约束**

- 单边仓位上限、总仓位上限、日亏损上限
- 每秒下单频率限制
- 异常订单自动取消机制

### 阶段四：高阶能力（区分普通做市商和顶级做市商）

**12. 信号接入（Alpha 层）**

- 短期价格预测：OBI、价格动量、跨所 lead-lag
- 接入方式：Cartea-Jaimungal 框架，把 μ 嵌入保留价
- 信号的实时 IC（信息系数）监控

**13. 参数的在线学习**

- γ 的自适应调整：根据实际库存路径和 P&L 反馈调整
- σ、A、k 的状态切换：识别市场制度（regime）变化
- 强化学习或贝叶斯优化的应用

**14. 多资产联合做市**

- 多个相关品种的协方差矩阵建模
- 组合层面的库存风险（相关品种可互相对冲）
- CJ 多资产框架的实战实现

**15. 跨场所套利 + 做市一体化**

- 做市本身可以制造跨所价差机会
- 在多个场所同时挂单，享受第一时间发现价差的优势
- 把套利收益和做市点差合并优化
